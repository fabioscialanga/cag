# Code of Conduct

## Our Standard

We want CAG to be a respectful, technically honest, and useful project space.

Examples of behavior that help:
- being constructive and specific
- assuming good intent while staying clear about problems
- giving actionable feedback
- welcoming different experience levels

Examples of unacceptable behavior:
- harassment or personal attacks
- dismissive or hostile communication
- doxxing or sharing private information
- deliberately disruptive behavior

## Scope

This applies to project discussions, issues, pull requests, and other public interactions around the repository.

## Enforcement

Project maintainers may remove, edit, or reject comments, issues, or contributions that violate this code of conduct.

## Reporting

If you experience or witness unacceptable behavior, open a private contact request through GitHub without sharing sensitive details publicly, and the maintainers can move the conversation to a private channel.
