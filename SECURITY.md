# Security Policy

## Supported Status

This repository is currently a **preview** project. Security hardening is improving, but the codebase should not be treated as production-grade infrastructure by default.

## Reporting a Vulnerability

Please do **not** post sensitive security details in a public issue.

Preferred path:
- use GitHub private reporting or a security advisory if available

Fallback path:
- open a minimal public issue such as `Security contact requested`
- do not include exploit details
- the maintainers can then move the report to a private channel

## What Helps in a Report

- affected area
- reproduction steps
- impact
- whether credentials, file access, or remote execution are involved

## Scope Notes

Preview-specific limitations, misconfiguration, and local setup failures are not always security vulnerabilities, but we still appreciate responsible disclosure when the impact is unclear.
